URIって

URLとURIって違うの？

違います。

URL(Uuniform Resource Locater)

=リソースの場所を示す

URI(Uniform Resource Identifier)

=リソースを特定する識別子

URIって

URIの中にhttp:やftp:などのスキームが存在
そのうちのLocationを示せるスキームがURL

URIって

 http://name:pass@hoge.jp:80/i/index.cgi?page=2&cate=3#fragment

URIの構成要素を見てみる

  • scheme httpとかhttpsとかftpとかetcetc...
  • autholrty 接続先の情報
    • userinfo ユーザ名とパスワード。多くのHttpは問われないので省略
    • host サーバのホスト名
    • port 使用するポート
  • path オーソリティの中のアクセス先
  • query さらに細かい指示
  • fragment 通常サーバには送られずクライアントで処理される

httpって

HyperTextTransferProtocol超文件伝輸協定

ハイパーなテキスト（HTML、XML etc）を転送するプロトコル(規定)

  • サーバ・クライアントで動作
  • テキストに限らず画像・動画等のバイナリも扱える
  • SSLで暗号化されるとHTTPS

httpって

歴史

  • HTTP/0.9(仕様が決まる以前の便宜的なバージョン)

    最初の頃はヘッダとかもなかった

  • HTTP/1.0

    1996年に正式なプロトコル仕様書が発行
    だが時は世紀末・各ベンダが独自実装してしまったために形骸化

  • HTTP1.1

    厳格な仕様・高度な機能
    インターネットの標準化

httpって

歴史

  • HTTP/1.2〜

    開発中止

    • 進化の過程で複雑になりすぎた
    • 1.1を拡張していく形ですることでこれからもよろしく

  • Version HTTPのバージョン
  • StatusCode 状態を表す3桁の数字
  • StatusMessage

ステータスコード

状態により1xx〜5xxの数値が返される

  • 1xx

    一時的なレスポンス。処理中。

  • 2xx

    成功

  • 3xx

    リダイレクション

  • 4xx

    クライアント側のエラー。リクエストが不正、URLを見つけ出せなかった等

  • 5xx

    サーバ側のエラー。予期せぬエラー、過負荷等

代表的なステータスコード

200

成功。レスポンスの中身はリクエストのメソッドによって異なる

301

リソースが移動したことを示す。

400

不正なリクエスト。リクエストの構文がおかしい

401

認証が必要

403

サーバ側が実行を拒否。実行権限のない動作を行った時とか

404

リソースが存在しない

500

サーバエラー。スクリプトのミスやらハードエラーやらいろいろ

503

サーバの都合でリクエストを処理できない。過負荷、メンテ等
